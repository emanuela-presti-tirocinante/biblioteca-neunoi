Indicazioni tecniche
STACK TECNOLOGICO 
- Frontend: React, con React Router per navigazione
- Backend: Node.js con Express
- Database: MySQL
- Autenticazione: JWT (JSON Web Token) con bcrypt per hash password 

FASE 1 - SETUP E STRUTTURA BASE:
1. Configurazione progetto React (con Vite o Create React App)
2. Configurazione server Express con struttura cartelle
3. Setup connessione MySQL con creazione database
4. Sistema di variabili ambiente (.env) per backend
5. Configurazione CORS per comunicazione frontend-backend

FASE 2 - DATABASE E MODELLI:
6. Creazione schema database MySQL con tabelle:
   - users (id, email, password_hash, nome, cognome, ruolo [user/admin], created_at, updated_at)
   - books (id, titolo, autore, genere, isbn, cod_archivio, descrizione, copertina_url, copie_totali, copie_disponibili, anno_pubblicazione, created_at)
   - categories (id, nome, immagine_url, created_at)
   - book_categories (relazione many-to-many: book_id, category_id)
   - loans (id, user_id, book_id, stato [pending/active/completed/rejected], data_richiesta, data_approvazione, data_inizio_prestito, data_scadenza, data_restituzione, created_at)
   - reviews (id, user_id, book_id, rating, testo, created_at)
   - quotes (id, testo, autore, created_at) per citazioni homepage ed email
7. Script di migration per creare le tabelle
8. Script di seed per importare i dati reali da file Excel che fornirò
   (contiene libri reali già organizzati per categorie)

FASE 3 - AUTENTICAZIONE:
8. API registrazione utente (POST /api/auth/register) con validazione email unica e hash password
9. API login (POST /api/auth/login) con generazione JWT
10. Middleware autenticazione per proteggere route
11. Middleware controllo ruolo admin
12. Context React per gestione stato autenticazione frontend
13. Componenti PrivateRoute e AdminRoute per proteggere pagine

FASE 4 - API BACKEND (REST):
14. Definisci e implementa tutte le API necessarie:

PUBBLICHE:
- GET /api/books - lista libri con paginazione, filtri, ricerca
- GET /api/books/:id - dettaglio singolo libro
- GET /api/categories - lista categorie
- GET /api/categories/:id/books - libri di una categoria
- GET /api/quotes/random - citazione casuale per homepage

UTENTE AUTENTICATO:
- POST /api/loans/request - richiesta prestito (controlla copie disponibili)
- GET /api/loans/my-loans - prestiti dell'utente (attivi, in attesa, storico)
- POST /api/reviews - crea recensione
- GET /api/users/profile - profilo utente
- PUT /api/users/profile - modifica profilo

ADMIN:
- GET /api/admin/loans - tutti i prestiti con filtri per stato
- PUT /api/admin/loans/:id/approve - approva prestito (aggiorna copie_disponibili)
- PUT /api/admin/loans/:id/reject - rifiuta prestito
- PUT /api/admin/loans/:id/complete - segna come restituito (incrementa copie)
- CRUD completo libri: POST/GET/PUT/DELETE /api/admin/books
- CRUD completo categorie: POST/GET/PUT/DELETE /api/admin/categories

FASE 5 - SISTEMA EMAIL:
15. Email automatiche da implementare:
    - Conferma registrazione con link verifica 
    - Notifica approvazione prestito con dettagli e data scadenza
    - Notifica rifiuto prestito
    - Promemoria scadenza (cron job: 3 giorni prima)
    - Notifica libro disponibile (se qualcuno era in attesa)
    - Email settimanale/mensile con citazione letteraria (cron job)
16. Sistema di template email riutilizzabili
